# RPG Maker 2000 遊戲在地化標準工作流程 (SOP)

本指南基於《Abyss》專案的實踐經驗，旨在為老舊遊戲引擎（如 RM2K/2K3）提供一套穩定、自動化且具備高度兼容性的在地化工作流程。

---

## 核心技術要求
- **編碼格式**：所有輸出檔案必須強制使用 `UTF-8-BOM`。
- **標籤保護**：絕對禁止修改引擎標籤結構（如 `#Message#`, `#Choice#`, `Select Face Graphic`）。
- **術語一致性**：建立字典（Glossary），確保地名、人名、道具名在全域一致。

---

## 階段一：準備與環境建置
1. **目錄結構**：
   - `StringScripts_Origin/`：存放原始日文檔案。
   - `StringScripts/`：存放工作中的翻譯檔案。
   - `segments/`：存放分割後的暫存檔。
   - `scratch/`：存放輔助腳本與分析結果。

2. **基礎掃描**：
   - 使用腳本統計所有檔案的標籤數量，作為最終校對的基準。

---

## 階段二：檔案分割與批次翻譯
當單一腳本檔案超過 500 行時，必須執行以下流程：

1. **自動分割 (`split_file.py`)**：
   - 將檔案按 500 行左右的長度分割，但**必須確保分割點不在 `#Message#` 塊的中間**。
2. **批次翻譯**：
   - 每 10 個分割檔完成後，必須進行一次中間校對。
   - 翻譯原則：保留前導空格，保留所有 `\N[x]`, `\C[x]` 等代碼。
3. **自動合併 (`merge_file.py`)**：
   - 將翻譯後的分割檔按原序合併。

---

## 階段三：品質保證與修正 (QA)
這是專案最關鍵的一步。

### 1. 標籤完整性檢查 (`final_check.py`)
- **邏輯**：對比 `Origin` 與 `Trans` 檔案中 `#Message#`, `#Choice#`, `Select Face Graphic` 的數量。
- **處理**：若數量不符，則必須回捲並重構該檔案。

### 2. 字元與標點清理 (`global_clean.py`)
- **清理對象**：
  - 日文長音符號 `ー` $\rightarrow$ 繁體中文破折號 `—`。
  - 日文中間點 `・` $\rightarrow$ 繁體中文點 `．`。
  - 日文句號 `。` $\rightarrow$ 繁體中文句號 `。` (確保寬度正確)。
- **日文殘留檢測**：搜尋 `[\u3040-\u309F\u30A0-\u30FF]` 範圍內的字元，確保無假名殘留。

### 3. 重構填充策略 (Refactoring)
針對結構損壞嚴重的檔案（如 `Map0010`）：
- 讀取原始檔案的標籤骨架。
- 程式化地將翻譯內容依序填入對應的 `#Message#` 區塊。
- 這樣能確保即使對白行數改變，引擎結構依然穩固。

---

## 階段四：最終驗證與交付
1. **全域掃描**：生成 `final_global_report.txt`，確認所有檔案狀態均為 `OK`。
2. **編碼確認**：確認所有檔案皆有 `BOM` 頭。
3. **實機測試**：將檔案覆蓋至遊戲目錄，測試重點為「地圖切換」與「長對話顯示」。

## 關鍵輔助腳本參考

### `final_check.py` 核心邏輯 (Python)
```python
def count_tags(filepath):
    tags = ["#Message#", "#Choice#", "Select Face Graphic"]
    counts = {t: 0 for t in tags}
    with open(filepath, 'r', encoding='utf-8-sig') as f:
        for line in f:
            for t in tags:
                if t in line: counts[t] += 1
    return counts
```

### `global_clean.py` 替換規則
```python
replacements = {
    'ー': '—',
    '・': '．',
    '～': '～',
    '？': '？',
    '！': '！'
}
```

## 注意事項
- **特殊名稱**：遇到如 `\N[1]` 被誤改為 `\n[1]` 或 `[N1]` 時，必須全域修正，否則遊戲內會顯示錯誤或無法獲取主角名。
- **角色稱呼**：注意「莎莉亞」與「酷」在不同檔案中的稱呼變化，保持譯名統一。
